# Contributing Guidelines

Thank you for contributing to this repository.

---

# Contribution Rules

Contributors should:

- Use clear commit messages
- Follow repository structure
- Write proper markdown formatting
- Keep documentation educational and legal
- Avoid harmful or illegal content

---

# Folder Guidelines

| Folder | Purpose |
|---|---|
| docs/ | Technical documentation |
| notes/ | Research notes |
| resources/ | References and links |
| scripts/ | Educational code examples |

---

# Pull Requests

Before submitting a pull request:

1. Check formatting
2. Verify accuracy
3. Ensure documentation quality
4. Follow ethical guidelines

---

# Reporting Issues

Use GitHub Issues for:
- documentation errors
- formatting problems
- broken links
- suggestions

---

# Code Standards

- Use readable filenames
- Add comments when necessary
- Keep scripts educational
- Avoid malicious content

---

# Thank You

Contributions help improve the quality of educational cybersecurity research.
