# Encryption Fundamentals

# Introduction

Encryption is the process of converting readable information into protected data using mathematical algorithms.

Encrypted data can only be accessed using the correct decryption key.

---

# Purpose of Encryption

Encryption protects:
- personal information
- financial data
- communications
- stored files
- network traffic

---

# Types of Encryption

# 1. Symmetric Encryption

Uses the same key for:
- encryption
- decryption

## Examples

- AES
- DES

## Advantages

- Fast performance
- Efficient for large data

## Limitations

- Key sharing challenges

---

# 2. Asymmetric Encryption

Uses:
- public key
- private key

## Examples

- RSA
- ECC

## Advantages

- Secure key exchange
- Digital signatures

## Limitations

- Slower than symmetric encryption

---

# Hashing

Hashing converts data into fixed-length values.

Examples:
- SHA-256
- SHA-3

Hashing is commonly used for:
- password storage
- integrity verification

---

# Encryption in Daily Life

Encryption protects:
- HTTPS websites
- messaging apps
- online banking
- cloud storage

---

# HTTPS

HTTPS encrypts communication between browsers and websites.

Benefits:
- prevents interception
- improves security
- protects login information

---

# End-to-End Encryption

Only sender and recipient can read messages.

Used by:
- Signal
- WhatsApp
- Session

---

# Common Encryption Risks

- Weak passwords
- Poor key management
- Outdated algorithms
- Human error

---

# Best Practices

# Use Modern Algorithms

Prefer strong standards like AES-256.

---

# Protect Encryption Keys

Keys should remain confidential.

---

# Update Software Regularly

Security patches fix vulnerabilities.

---

# Enable Full Disk Encryption

Protects devices from unauthorized access.

---

Encryption is widely used for:
- security
